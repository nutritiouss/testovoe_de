# Тестовое задание: Data Engineer / Analytics Engineer

   Организация занимается рассылкой международных СМС. Дана Витрина SMS-сообщений в ClickHouse. Одна строка — одно SMS.  В конце представлено описание полей таблицы. Витрина собирается с различных потоков данных (flow), но для реализации тестового задания этап сборки был пропущен и сокращено количество столбцов. Витрину использует не только Superset, но и клиенты через API. Поэтому будьте внимательны с ключами сортировки и партиционирования.

### Твоя задача:

- поднять в docker  ClickHouse, Apache Superset. 
- спроектировать витрину данных (список полей дан ниже) и сформировать DDL ClickHouse
- написать генератор данных. Временной интервал 1 год, общее количество 1 млн строк. 
- собрать дашборд **SMS Operations** в Apache Superset 
- сделать скриншоты дашбордов
- объяснить как происходил выбор ключа сортировки, партиционирования/индексов/движков

### Можно сделать дополнительно

- Кастомизировать Superset через Feature flags
- добавить DQ дашборд для владельца данных
- развернуть ClickHouse+CH keeper 3 nodes в кластерном режиме и поправить DDL.

### Было бы круто:

- сделать проект воспроизводимым. Проект должен запускаться  с команды `docker compose up -d` запускается создание таблицы, генерация данных, создание дашборда в superset. Приветствуется Makefile. Объекты Superset нужно хранить в YAML, чтобы трекать изменения дашбордов через git

## Дашборд Apache Superset

Соберите дашборд **SMS Operations** на данных `messages_mart` в ClickHouse. Ниже приведен возможный вариант реализации дашборда. Приветствуется применение собственных метрик, типов чартов и т.д.

### Глобальные фильтры (минимум)


| Фильтр | Поле                 |
| ------ | -------------------- |
| Клиент | `customer_id`        |
| Период | диапазон `sent_date` |


### Обязательные чарты (минимум)


| Чарт           | Тип                        | Логика                                                 |
| -------------- | -------------------------- | ------------------------------------------------------ |
| SMS по дням    | Time series                | `count(message_id)` по дню `sent_date`                 |
| Delivery rate  | Big number                 | доля `delivery_status = 'DELIVRD'` от общего числа SMS |
| Топ стран      | Bar                        | `count(message_id)` по `country`, top 10               |
| Топ операторов | Bar / Pie                  | `count(message_id)` по `receiver_operator`             |
| Выручка        | Time series или Big number | `sum(price)` по каждой валюте                          |


### На усмотрение кандидата

- дополнительные фильтры: `country`, `delivery_status`, `currency`
- чарты: failed statuses, `delivery_time`, `segment_count`, inbound/outbound (`direction`)
- исключение soft-delete: `WHERE deleted_at IS NULL`

***По дашбордам нет жесткого ТЗ. Приветствуются любые ваши идеи и их реализации.***

# Описание полей

### Идентификаторы


| Поле               | Тип      | Описание                                                                                    |
| ------------------ | -------- | ------------------------------------------------------------------------------------------- |
| `customer_id`      | `UInt32` | Идентификатор клиента. Достаточно 5-10 клиентов                                             |
| `application_uuid` | `UUID`   | UUID приложения клиента, привязанного к `customer_id`. Достаточно 1-4 приложения на клиента |
| `message_id`       | `UUID`   | Идентификатор SMS на стороне платформы / вендора.                                           |


### Время и адресация


| Поле            | Тип                                | Описание                                                                        |
| --------------- | ---------------------------------- | ------------------------------------------------------------------------------- |
| `sent_date`     | `DateTime64(6)`                    | Дата/время отправки SMS.                                                        |
| `sender`        | `LowCardinality(Nullable(String))` | Альфа-имя отправителя (бренд): `Google`, `Amazon`, `PayPal` и т.д.              |
| `receiver`      | UInt64                             | Номер получателя в международном формате (MSISDN). Префикс зависит от `country` |
| `country`       | `LowCardinality(Nullable(String))` | Страна направления                                                              |
| `segment_count` | `UInt32`                           | Количество сегментов                                                            |


### Статусы доставки


| Поле              | Тип                                | Описание                                                                                                                                              |
| ----------------- | ---------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| `delivery_status` | `LowCardinality(Nullable(String))` | Итоговый статус доставки. Примеры: `DELIVRD`, `UNDELIV`, `EXPIRED`, `REJECTD`, `SENT`, `NO ROUTES`, `ROUTE FAILED`; полный список — в data_spec.json. |
| `attempt_number`  | `UInt8`                            | Номер попытки доставки сообщения.                                                                                                                     |
| `delivery_time`   | `UInt16`                           | Время доставки в миллисекундах.                                                                                                                       |


### Тарификация и направление


| Поле                | Тип                                | Описание                                            |
| ------------------- | ---------------------------------- | --------------------------------------------------- |
| `price`             | `Float32`                          | Цена SMS для клиента (в валюте `currency`)          |
| `currency`          | `LowCardinality(Nullable(String))` | Валюта тарифа: `EUR`, `USD`,`RUB`.                  |
| `receiver_operator` | `LowCardinality(Nullable(String))` | Оператор получателя                                 |
| `direction`         | `UInt8`                            | Направление трафика: `0` — inbound, `1` — outbound. |


### Метаданные


| Поле         | Тип                       | Описание                       |
| ------------ | ------------------------- | ------------------------------ |
| `created_at` | `DateTime64(6)`           | Время создания записи.         |
| `updated_at` | `DateTime64(6)`           | Время последнего обновления.   |
| `deleted_at` | `Nullable(DateTime64(6))` | Время soft-delete в источнике. |
